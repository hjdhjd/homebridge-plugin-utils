# webui-loader

The webUI script-loader stamp. A plugin's `index.html` hand-copies an importmap/cache-bust `<script type="module">` block whose runtime job is to read the
homebridge-plugin-utils manifest, inject an importmap mapping the bare package specifier to the hashed-versioned subdir the `prepare-ui` CLI mirrors into place, and
dynamically import the plugin's entry module. That block is identical across the family, so this module renders it from one template: the plugin declares its entry
and cache-bust list in a config comment, and `prepare-ui` stamps the rendered block into a marker-fenced region on every build.

Three pieces: the marked-region marker constants (following the family's uniform template - see `docChrome.ts`), [parseWebUiLoaderConfig](#parsewebuiloaderconfig) which reads and
validates the plugin's config comment, and [renderWebUiLoaderScript](#renderwebuiloaderscript) which renders the loader block from that config plus the destination and package facts
the CLI supplies. The CLI's `prepareUi` owns the read/splice/atomic-write around them.

## WebUI Loader

### WebUiLoaderConfig

The validated shape of a plugin's webUI loader config: the entry module the loader dynamically imports, and the list of hand-authored modules (the entry aside) that
sit outside the package's hashed versioning and therefore each need a cache-bust.

#### Properties

| Property | Modifier | Type |
| ------ | ------ | ------ |
| <a id="bust"></a> `bust` | `readonly` | readonly `string`[] |
| <a id="entry"></a> `entry` | `readonly` | `string` |

### parseWebUiLoaderConfig()

```ts
function parseWebUiLoaderConfig(html, htmlPath): WebUiLoaderConfig;
```

Read and validate the plugin's webUI loader config comment from an `index.html`. The comment is the plugin's sole declaration - its entry module and its cache-bust
list - and everything else about the rendered loader is a family constant or a fact the CLI supplies, so this is where a mistake in that declaration surfaces with a
message naming the file and the defect.

Rejects, each with a framed error naming `htmlPath`: an absent config comment, a duplicated one (the region it drives would be ambiguous), an unterminated one,
invalid JSON, a mis-shaped object (a missing/non-string `entry`, or a `bust` that is not a string array), and a config comment sitting INSIDE the marked region (the
first stamp would erase it, so it must live outside the marker pair).

#### Parameters

| Parameter | Type | Description |
| ------ | ------ | ------ |
| `html` | `string` | The full `index.html` text. |
| `htmlPath` | `string` | The path the html was read from, named in every diagnostic. |

#### Returns

[`WebUiLoaderConfig`](#webuiloaderconfig)

The validated [WebUiLoaderConfig](#webuiloaderconfig).

#### Throws

`Error` naming `htmlPath` and the defect for an absent, duplicated, unterminated, invalid-JSON, mis-shaped, or inside-the-region config comment.

### renderWebUiLoaderScript()

```ts
function renderWebUiLoaderScript(args): string;
```

Render the webUI script-loader block from the plugin's config plus the destination and package facts the CLI supplies. The rendered block reproduces the proven
runtime semantics of the hand-authored loader: a single `Date.now()` cache-bust stamp shared across the page load, a `bust()` helper that resolves a path against
`import.meta.url` and appends the stamp, a `cache: "no-store"` fetch of the manifest, an importmap, and a dynamic import of the entry.

Every importmap module entry keeps the BARE module path as its KEY with the busted URL as its VALUE - the entry module ITSELF plus every declared `bust` entry, so
the most important module (the entry) is never served stale. The trailing-slash `<packageName>/` prefix entry maps the bare package specifier to the hashed-versioned
subdir, and transitive imports inherit it through relative-URL resolution.

`libPath` is the destination-relative segment the CLI derives from its `dest` basename (`"./lib/"` for the family convention), so the manifest fetch path and the
prefix URL are path-correct for any destination rather than a hardcoded `./lib/` that is only coincidentally right for one plugin.

#### Parameters

| Parameter | Type | Description |
| ------ | ------ | ------ |
| `args` | \{ `bust`: readonly `string`[]; `entry`: `string`; `libPath`: `string`; `packageName`: `string`; \} | - |
| `args.bust` | readonly `string`[] | The hand-authored modules (the entry aside) that each need a cache-bust. |
| `args.entry` | `string` | The entry module the loader dynamically imports; it is cache-busted alongside the `bust` list. |
| `args.libPath` | `string` | The destination-relative segment (e.g. `"./lib/"`) the manifest fetch and the importmap prefix resolve against. |
| `args.packageName` | `string` | The mirrored package's name; the importmap prefix maps `<packageName>/` to the hashed-versioned subdir. |

#### Returns

`string`

The `<script type="module"> ... </script>` block to splice between the marker pair.
